Introduction: Purpose and Scope

This document provides comprehensive explanations of every design, branding, and strategic concept underlying FAI's current brand system. It's designed to enable you to:

  • Understand the theoretical foundations of FAI's visual identity
  • Make informed design decisions consistent with the established system
  • Extend the brand system to new applications and contexts
  • Communicate effectively with design vendors and collaborators
  • Evaluate design work against established principles

This guide is organized into major sections covering brand strategy, design methodology, typography, color theory, visual design systems, implementation, and specific application to FAI.

Accessibility & Implementation

WCAG 2.1 Accessibility Requirements
▼

Color Contrast Requirements

  • WCAG AA (Minimum): 4.5:1 contrast for normal text, 3:1 for large text
  • WCAG AAA (Enhanced): 7:1 contrast for normal text, 4.5:1 for large text
  • Large text: 18pt+ or 14pt+ bold

Guidelines for FAI Colors

  • International Orange (#EE6133): Meets WCAG AA on white background (5.2:1)
  • Use dark text on light backgrounds: Ensures readability for all users
  • Don't rely on color alone: Use patterns, icons, or text labels to indicate status
  • Test with color blindness simulators: ~8% of men have red-green color blindness

Typography Accessibility

  • Minimum font size: 16px on screen, 12px in print
  • Line height minimum: 1.5 for body text
  • Line length: 50-80 characters for optimal readability
  • Use semantic HTML: Use proper heading levels (h1, h2, h3, etc.)
  • Keyboard navigation: All interactive elements must be keyboard accessible

For detailed WCAG 2.1 guidelines, see: WCAG 2.1 Quick Reference

Implementing FAI's Design System

The concepts outlined in this guide form the foundation for FAI's visual identity. Key takeaways for anyone working with the FAI brand:

For Designers: Understand the theoretical foundation behind every choice. Use grids consistently, respect the type scale, and apply the color system strategically. Your work should feel systematic and modern, never decorative.
For Brand Managers: Maintain centralized governance over logo usage and high-visibility materials, but empower teams with templates and clear guidelines. Regular audits ensure the system stays current and consistent.
For Developers: Implement typography carefully using the pentatonic scale. Ensure sufficient contrast ratios for all text. Use semantic HTML and keyboard-navigable interfaces. Test with accessibility tools.

Next Steps

  1. Review this guide regularly and share with team members
  2. Create design templates embedding the system (PowerPoint, Figma, etc.)
  3. Conduct quarterly audits of all public-facing materials
  4. Update guidelines based on new use cases and lessons learned
  5. Train new team members using this conceptual foundation

Questions? Refer back to this guide for theoretical context. When creating design work, ask: "Does this follow FAI's systematic approach? Is it modern, clear, and accessible?"
